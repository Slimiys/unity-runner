## Sandbox Runner

Небольшой учебный 3D‑раннер на Unity (URP), демонстрирующий базовую архитектуру по принципам **SOLID** и компонентный подход Unity.

### Возможности

- **Автоматическое движение вперёд**: персонаж постоянно бежит по оси Z.
- **Управление по горизонтали**: смещение влево/вправо по оси X.
- **Прыжок и проверка касания земли**.
- **Сбор монет** и подсчёт очков.
- **Зона падения** с респауном игрока в стартовую позицию.
- **Камера‑следилка** с настраиваемым смещением.

### Требования

- Unity **2022.3 LTS** или новее (URP‑шаблон, IL2CPP для Android).
- TextMesh Pro (подключается автоматически при открытии сцены).

### Структура проекта

- `Assets/Scenes` — игровые сцены (основная сцена: `SampleScene`).
- `Assets/Scripts/Core` — базовая логика:
  - `PlayerController` — движение игрока и прыжок.
  - `CameraFollow` — слежение камеры за игроком.
  - `IRespawnable` — интерфейс для объектов, поддерживающих респаун.
- `Assets/Scripts/Gameplay` — геймплейные сущности:
  - `Coin` — монетка, которую игрок может собирать.
  - `CoinSpawner` — спавнер монет в заданной области.
- `Assets/Scripts/World` — логика окружения:
  - `DeathZone` — зона, возвращающая игрока в стартовую позицию.
- `Assets/Scripts/UI` — интерфейс:
  - `ScoreManager` — отображение и учёт очков.
- `docs` — дополнительная документация по проекту.

### Управление (по умолчанию, ПК)

- Движение влево/вправо: **A / D** или стрелки (оси горизонтального ввода).
- Ускорение вперёд: **W** (увеличивает скорость вперёд до максимального множителя).
- Замедление: **S** (уменьшает скорость вперёд до минимального множителя).
- Прыжок: **Space** (если персонаж находится на земле).

Управление основано на новой системе ввода Unity (`Input System`), действия описаны в файле `Assets/PlayerControls.inputactions`.

### Запуск проекта

1. Откройте проект в Unity Hub (путь к корню репозитория).
2. Убедитесь, что установлен модуль **Universal Render Pipeline**.
3. Откройте сцену `Assets/Scenes/SampleScene.unity`.
4. Нажмите **Play** для запуска раннера в редакторе.

Для сборки под Android или другую платформу настройте параметры в `Project Settings → Player` и выполните обычный билд.

### Документация

Вся дополнительная документация находится в папке `docs`:

- `docs/CodingStandards.md` — правила кодирования и архитектурные договорённости (SOLID, структура папок, соглашения по именованию и документации).

План развития раннера (GDD, дизайн уровней, механики) рекомендуется оформлять отдельными файлами в этой же папке (`docs/RunnerPlan.md`, `docs/GameDesign.md` и т.п.).

### Архитектурные принципы

- Следование принципам **SOLID** и компонентному подходу Unity.
- Разделение ответственности по областям:
  - Ввод и движение игрока.
  - Генерация и логика геймплейных объектов (монеты, препятствия и т.п.).
  - Окружение и управление жизненным циклом (зоны, респаун).
  - UI и отображение состояния (очки, HUD).
